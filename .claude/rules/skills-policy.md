\# Skills Policy



\## Internal skills

\- project-architect

\- feature-prioritizer

\- database-designer

\- api-designer

\- permission-modeler



These are the default project skills.



\## External skills

\- find-skills

\- web-design-guidelines

\- webapp-testing

\- frontend-design



\## Usage rules

\- do not use skills by default

\- first use project context and internal skills

\- use external skills only if a real capability is missing

\- maximum 1 external skill per task

\- maximum 2 only if strongly justified



\## Special rules

\### find-skills

Use only if no existing skill covers the need.

Suggest at most 2 skills and choose only 1 unless necessary.



\### frontend-design

Use only for limited UI inspiration.

Do not redesign the entire app.

Do not make the project overly polished or complex.



\### webapp-testing

Use only after core features exist.

Prefer V2 or later.



\## Forbidden behavior

\- do not stack many external skills

\- do not add complexity because a skill allows it

\- do not override project constraints

\- do not introduce React, Vue, Next.js, or similar frameworks
